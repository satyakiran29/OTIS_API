# Contributing to OTIS API

First off, thank you for considering contributing to OTIS API! It's people like you that make OTIS a great tool for Temple Management.

### 1. Where do I go from here?

If you've noticed a bug or have a feature request, make one! It's generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

If you'd like to start coding, check out out the open issues.

### 2. Fork & create a branch

If this is something you think you can fix, then fork OTIS_API and create a branch with a descriptive name.

### 3. Implement your fix or feature

At this point, you're ready to make your changes. Feel free to ask for help!
Everyone is a beginner at first.

### 4. Make a Pull Request

At this point, you should open a Pull Request.

- State clearly what your PR achieves.
- If it fixes a bug, link it.
- If it's a new feature, explain why it should be implemented.
